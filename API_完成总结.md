# ✅ API 文档整理说明

## 概述

这份文件不再承担“接口完成清单”或“功能发布公告”的职责。

原因很简单：

- 接口数量和实现状态会持续变化
- 如果这里再维护一份接口列表，极容易和实际代码漂移
- 仓库里已经有更适合的文档分别承担接口定义、集成说明和使用示例

现在，这份文件只用于说明各份 API 文档的分工，帮助读者快速找到正确入口。

---

## 文档分工

### 1. 接口定义

看这里：

- [API_接口文档.md](API_接口文档.md)

适合查询：

- 接口路径
- 请求参数
- 返回结构
- 错误码
- 数据单位
- 版本更新

这是 API 的权威说明入口。

### 2. 集成接入

看这里：

- [API_集成指南.md](API_集成指南.md)

适合查询：

- 怎么接入服务
- 推荐使用哪些接口组合
- 超时与重试建议
- 常见接入方案
- 对接排坑

这份文档强调的是“怎么用”，不是“字段字典”。

### 3. 示例代码

看这里：

- [API_使用示例.py](API_使用示例.py)

适合：

- 快速验证服务是否可用
- 参考 Python 调用方式
- 做二次脚本改造

### 4. Codex Skill

如果你在支持 skill 的 Codex 环境里工作，可使用：

- [skills/tdx-api-docs/SKILL.md](skills/tdx-api-docs/SKILL.md)

适合查询：

- 某个接口是否真的已实现
- 文档与代码是否一致
- 某个字段实际叫什么
- HTTP / WebSocket 调用方式

---

## 当前建议

建议统一遵循下面的使用顺序：

1. 先看 [API_接口文档.md](API_接口文档.md)
2. 需要接入建议时看 [API_集成指南.md](API_集成指南.md)
3. 需要跑样例时看 [API_使用示例.py](API_使用示例.py)
4. 需要核对实现细节时看代码或使用 `tdx-api-docs` skill

---

## 维护原则

后续如果新增、修改或废弃接口：

- 优先更新 [API_接口文档.md](API_接口文档.md)
- 接入方式变化时同步更新 [API_集成指南.md](API_集成指南.md)
- 示例失效时更新 [API_使用示例.py](API_使用示例.py)
- 不再在本文件中维护完整接口清单

这样可以减少重复维护，降低文档漂移风险。
